AdsNative RenderJS

AdsNative provides various options to deliver native ad on your websites.

'Generalized' RenderJS (default behaviour)

Following is default ad tags provided by AdsNative, which will automatically render configured placement or native ad at right location on intended pages. It's called "Generalized" RenderJS as you can use same piece of code for any placement and site domain you have, making it super easier for publishers to install the tags on their sites. Insert the following tags in your webpage in <head> section of your site i.e. just before </head> tag.

<!-- Begin ADSNATIVE Code -->
<script type="text/javascript" src="http://static.adsnative.com/static/js/render.v1.js"></script>
<!-- End ADSNATIVE Code -->

Making RenderJS Effective

"Generalized" RenderJS uses configured CSS Path to search and insert native ad at right location. This can sometimes add to latency to render the ad. If you think that seems to be the case, you can choose one of the following approaches to reduce that latency.

Option 1

In this option you can insert the following tags exactly at location where native ad should be rendered,

<!-- Begin ADSNATIVE Code -->
<script type="text/javascript">
var _AdsNativeOpts = {
	'zid': '<ADZONE_KEY>',
};
</script>
<script type="text/javascript" src="http://static.adsnative.com/static/js/render.v1.js"></script>
<!-- End ADSNATIVE Code -->

Option 2

In this option you can insert the following tags just before </body> tags. It requires you to specify the element id where native ad should be rendered. It is required that given element should be empty. When native ad is rendered, it will replace that element.

<!-- Begin ADSNATIVE Code -->
<script type="text/javascript">
var _AdsNativeOpts = {
    adUnits: [
	{
	'zid': '<ADZONE_KEY_1>',
	'nativeAdElementId': 'div-id-1'
	},
	{
	'zid': '<ADZONE_KEY_2>',
	'nativeAdElementId': 'div-id-2'
	},
]
};
</script>
<script type="text/javascript" src="http://static.adsnative.com/static/js/render.v1.js"></script>
<!-- End ADSNATIVE Code -->

RenderJS Advanced API

Requires beginner to intermidiate JavaScript chops

In some advanced cases you may not want AdsNative RenderJS to handle automatic ad rendering, such as infinite scrolling, or if you want to show native ad on certain user actions, etc. In this case you may want circumvent RenderJS default behaviour. Insert following tag in HEAD section of your webpage. This way RenderJS won’t fetch or render any native ad until explicitly called by given functions. Please also note that you need to include above code before using any of the given function calls.

<!-- Begin ADSNATIVE Code -->
<script type="text/javascript">
var _AdsNativeOpts = {
    blockAdLoad: true
};
</script>
<script type="text/javascript" src="http://static.adsnative.com/static/js/render.v1.js"></script>
<!-- End ADSNATIVE Code -->

Prefetch Native Ad

You can choose to prefetch the native ad from AdsNative server for a given ad unit before the document is ready and also before you are ready to render the ad. This way there won’t be any latency for network request and ad will be for ready for rendering when it needs to.

All the calls to API are handled through ‘_AdsNative’ object.

var keywords = []
_AdsNative.fetchAds(['YOUR_PLACEMENT_KEY-1', 'YOUR_PLACEMENT_KEY-2'], keywords, function(status, apiKey){
    if(status){ //if a native ad is returned

    }
});

Above is one example of ‘fetchAds’ function will all available arguments. You can fetch multiple placements and you can also specify current page keywords for keyword targeting. You can notice that you can also provide callback for the event the ad request is finished. In the callback function it provides ‘status’ and ‘apiKey’ as arguments. ‘status’ is boolean value which indicates whether there is an ad returned for a current request and depending on that you can take an action. apiKey gives indication for which placement the call is for. The callback is returned for each placement.

Display Native Ad

Once the ad is prefetched you can choose to display that native ad at desired event. For example, you may want to display it when user scroll down in case of infinite scroll or show it when second page shows up, etc. You can use ‘displayAd’ function to show an ad for a placement.

_AdsNative.displayAd('YOUR_PLACEMENT_KEY-1', 'native-ad-element-id')

You need to specify the api key of the placement that needs to be displayed and then ID of the element in which the native ad should be displayed. This ‘native-ad-element-id’ element should be an empty div as upon displayAd this element will be replaced by the native ad.

Complete Example

Following is a complete example which use all above functions. It’s a simple example where ad gets displayed right away once the page is ready.

<html>
	<head>
	</head>
<body>
	<div id="an-ad-abc"></div>
	<div id="an-ad-xyz"></div>

	<!-- Begin ADSNATIVE Code -->
	<script type="text/javascript">
	var _AdsNativeOpts = {
	    blockAdLoad: true
	};
	</script>
	<script type="text/javascript" src="http://static.adsnative.com/static/js/render.v1.js"></script>
	<!-- End ADSNATIVE Code -->

	<script type="text/javascript">
	_AdsNative.fetchAds(['abc', 'xyz'], ['sports', 'autos'], function(status, apiKey){
	    if(status){ //if a native ad is returned
	        $(document).ready(function(){
	             _AdsNative.displayAd(apiKey, 'an-ad-'+apiKey)
	        });
	    }
	});
	</script>

</body>
</html>
